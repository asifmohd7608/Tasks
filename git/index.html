Git

git initialization

  • git init
  • git clone <url>
  • git remote add <remote_repo_name> <remote_repo_url>
  • git config --global user.name <name>
  • git config --global user.email <email>

git commit

  • git commit -m 'message'
  • multiline commits
    • git commit opens text editor
    • git commit -m "Commit Title" -m "Commit Description" each -m starts in a new line

git push

  • git push -u <remote_name> <local_branch_name>

git log

  • a utility tool to review and read a history of everything that happens to a repository.
  • git log
  • git log --oneline one commit per line.
  • git log --stat displays the files that have been modified, the number of lines that have been added or removed.
  • git log --after="yy-mm-dd" display all the commits made after the given date.

view a commit

  • git show <hash> view a specific commit

git compare

  • git diff --staged lists out the changes between the staged area and last commit.
  • git diff --staged <file_name> compares the changes of the specified file between staged area and last commit
  • git diff <branch_name1> <branch_name2> compare the changes from the first branch with changes from the second branch.
  • git diff <branch_name1> <branch_name2> <file_name> compares the changes of the specified file between two branches
  • git diff <commit_hash> <commit_hash> compares the changes between two commits.
  • git diff <commit_hash> <commit_hash> <file_name> compares the changes in a file between two commits.

git unstage file

  • git reset to unstage all files.
  • git reset <filename> to unstage a particular file only.

amend commits

  • to change commits.
  • git commit --amend

retrieve old versions

  • two methods
  • checkout method
    • git checkout <commit-hash> <filepath> this retrieves old version of the specified file.
    • git checkout <commit-hash> retrieves all files from the specified commit.
  • reset method
    • git reset <commit-hash> reverts to an old commit.

revert a commit

  • git revert <commit_hash>
  • reverts a change made in that particular commit.

remove untracked files

  • git clean -n dry run - shows all untracked files about to be removed.
  • git clean -f removes all untracked files.

git ignore

  • files or directories specified inside .gitignore will not be tracked or committed
