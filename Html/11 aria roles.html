Aria Roles

Aria roles provide semantic meaning to content, allowing screen readers and other tools to present and support interaction with object in a way that is consistent with user expectations of that type of object

There are 6 categories of aria roles

  1. Document structure roles : provides a structural description for a section of content. eg:- toolbar
  2. Widget roles : used to define common interactive patterns, rquires js interaction. eg:- searchbox
  3. Landmark roles : provides a way to understand the structure of the page. eg:- search
  4. live region roles : used to define elements whose contents will be dynamically changed. eg:- alert
  5. Window roles : used to define sub windows to the main document windows. eg:- modal
  6. Abstract roles : given by browsers
